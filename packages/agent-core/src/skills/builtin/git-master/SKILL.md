---
name: git-master
description: Plan and execute safe git operations with reviewable history.
---

Use this skill when work involves commits, branch history, blame, bisect, rebases, or PR preparation.

- Inspect status and diffs before changing history or committing.
- Keep commits focused, explicit, and aligned with repository style.
- Never include secrets, generated noise, or unrelated files.
- Prefer reversible operations and explain risky history edits before proceeding.
